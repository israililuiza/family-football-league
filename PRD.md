# ТЗ: Семейные задания и баллы

## 1. Цель
Веб-страница для семьи (Эдуард, Луиза, Адам, Ясмин, Авива), которая мотивирует выполнять домашние задания через систему баллов и наград.

## 2. Роли
- **Родитель / администратор** — добавляет и удаляет участников, задания, награды; запускает еженедельный сброс.
- **Участник** — отмечает задания выполненными, обменивает баллы на награды.

В MVP роли технически не разграничены: любой, у кого открыта страница, может выполнять любые действия. Разграничение прав — в разделе «Дальнейшие доработки».

## 3. Функции (реализовано в MVP)
1. Управление участниками: добавление, удаление, аватар-эмодзи, счётчик баллов.
2. Задания: название, стоимость в баллах, назначение конкретному участнику или «любому».
3. Отметка выполнения — начисляет баллы назначенному участнику, задание блокируется до сброса недели.
4. Рейтинг — сортировка участников по баллам с медалями за топ-3.
5. Награды: создание, стоимость в баллах, обмен (списывает баллы, если их достаточно).
6. Еженедельный сброс: обнуляет баллы всех участников и статус выполнения заданий, фиксирует дату последнего сброса.
7. Хранение данных — в localStorage браузера, без сервера.

## 4. Пользовательские сценарии
- Родитель добавляет задание «Помыть посуду» на 10 баллов, назначает Адаму.
- Адам выполняет задание и нажимает «Выполнено» → получает +10 баллов, растёт в рейтинге.
- Когда у Ясмин накопилось 50 баллов, она обменивает их на «Час игр на планшете».
- В понедельник родитель нажимает «Начать новую неделю» → баллы всех обнуляются, задания снова доступны.

## 5. Нефункциональные требования
- Работает офлайн, без бэкенда и регистрации.
- Адаптивный дизайн (мобильный и десктоп).
- Один HTML-файл (HTML/CSS/vanilla JS), без внешних зависимостей.

## 6. Ограничения MVP (вне рамок v1)
- Нет синхронизации между устройствами / облачного хранения — данные привязаны к конкретному браузеру.
- Нет разграничения прав доступа (PIN/пароль для родителя).
- Нет истории/архива по прошлым неделям — при сбросе данные не сохраняются.
- Нет уведомлений (push, email).

## 7. Дальнейшие доработки (backlog)
- Облачная синхронизация (например, Firebase/Supabase) для доступа с разных устройств.
- Архив истории по неделям и статистика прогресса.
- PIN-код для родительских действий (удаление, сброс, редактирование наград).
- Push- или email-напоминания о невыполненных заданиях.
- Фото/подтверждение выполнения задания.

## 8. Технический стек
- Frontend: HTML5, CSS3, vanilla JavaScript.
- Хранение: `localStorage` (ключ `familyTasksAppData`).
- Файл: `family-tasks.html` — открывается напрямую в браузере, установка не требуется.
